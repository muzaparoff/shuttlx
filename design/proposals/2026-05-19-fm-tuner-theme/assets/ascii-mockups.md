# FM Tuner вҖ” ASCII Mockups

All mockups use the following conventions:

- `в–Ҳ` = bright cyan `#7CD8FF` (lit LCD segment, primary text, hero digits)
- `в–“` = dim cyan `#0E6580` (unlit LCD segment, secondary text, dim chrome)
- `в–‘` = navy background `#021018`
- `в–Ј` = filled VU bar segment (cyan, lit)
- `в–ў` = empty VU bar segment (dim outline, unlit)
- `в—ўв—Ө` = red `#FF6B6B` destructive accent (stop only)
- `в–¶ в—Җ` = chrome arrows in header pill

The VU column at the left edge is drawn by the screen background modifier (Canvas), not the screen itself. Mockups show its visible position but assume it is rendered behind content.

## State variants вҖ” required for every dynamic screen

For each timer screen, the implementer must handle:

| State | Timer | VU bar | Step pill | Footer info | Buttons |
|-------|-------|--------|-----------|-------------|---------|
| **Idle** (pre-start) | `--:--` dim cyan | breathing pulse 0.3-0.5 | hidden | `READY В· NO SIGNAL В· TUNE STATION` | Single START (cyan) |
| **Loading** (HK auth, sensor warmup) | `--:--` dim cyan, blinking colon | static at 0.5 | hidden | `SEARCHING В· NO LOCK` | Disabled START |
| **Running** | bright cyan, live | `bpm / 200` live | bright cyan | live metrics | Pause + Stop |
| **Paused** | bright cyan, blinking colon | static at last value | dim cyan | `PAUSED В· SIGNAL HOLD` | Resume + Stop |
| **Error** (HK denied / no HR) | `--:--` dim cyan | absent | `в•‘ NO SIGNAL в•‘` red `#FF6B6B` | `ERROR В· CHECK PERMISSIONS В· TAP SETTINGS` | Single SETTINGS button |
| **Empty** (Programs with no templates) | n/a | absent | n/a | `NO STATIONS PROGRAMMED` | Single "CREATE PROGRAM" |

The blinking colon during paused/loading: 1.0 Hz, `.opacity(value ? 1.0 : 0.3)` toggled by `Timer.publish(every: 0.5)`.
